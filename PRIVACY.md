## 2. Збір даних

### 2.1 Серверна сторона

Програма **не здійснює жодного серверного збору даних**. Відсутні:

- серверні бази даних;
- аналітичні трекери (Google Analytics, Mixpanel тощо);
- зовнішні HTTP-запити до сторонніх API;
- механізми автентифікації або реєстрації користувачів.

### 2.2 Локальне сховище (LocalStorage)

Програма використовує `window.localStorage` браузера для збереження:

| Ключ                | Призначення                                  | Тип даних    |
|---------------------|----------------------------------------------|--------------|
| `wordSearchState`   | Налаштування гри (розмір сітки, кількість слів) та таблиця лідерів | JSON-об'єкт  |

- Дані зберігаються **виключно на пристрої користувача**.
- Дані **не передаються** на жодні зовнішні сервери.
- Користувач може в будь-який момент видалити ці дані через налаштування браузера або консоль розробника (`localStorage.clear()`).

### 2.3 Cookie

Програма **не використовує HTTP Cookie** для трекінгу або аналітики. Компонент Cookie Popup використовує LocalStorage для збереження стану згоди користувача (див. розділ 5).

## 3. Відповідність GDPR

Програма відповідає принципам Загального регламенту захисту даних (GDPR — General Data Protection Regulation) Європейського Союзу:

| Принцип GDPR                   | Реалізація в програмі                                                |
|---------------------------------|----------------------------------------------------------------------|
| **Мінімізація даних**          | Зберігаються лише налаштування гри та результати, без персональних даних |
| **Прозорість**                 | Ця політика описує всі механізми збереження даних                     |
| **Контроль користувача**       | Користувач може видалити всі дані через `localStorage.clear()`       |
| **Відсутність передачі третім сторонам** | Дані не покидають пристрій користувача                        |
| **Право на видалення**         | Повне видалення даних доступне в будь-який момент                    |

## 4. Права користувача

Користувач має повний контроль над своїми локальними даними:

1. **Перегляд** — відкрити DevTools → Application → LocalStorage.
2. **Видалення** — виконати `localStorage.removeItem('wordSearchState')` або `localStorage.clear()` у консолі браузера.
3. **Експорт** — скопіювати вміст LocalStorage з DevTools.

## 5. Політика Cookie (Cookie Policy)

### 5.1 Опис компонента Cookie Popup

Компонент `CookiePopup` — це UI-елемент, що інформує користувача про використання локального сховища браузера відповідно до вимог GDPR.

### 5.2 Механізм роботи

1. При першому відвідуванні сайту відображається банер Cookie Popup.
2. Користувач натискає кнопку **"Прийняти"** (`onAccept`).
3. Стан згоди зберігається в `localStorage` під ключем `cookieConsent` зі значенням `"accepted"`.
4. При наступних відвідуваннях компонент перевіряє значення `localStorage.getItem('cookieConsent')`:
   - Якщо `"accepted"` — банер **не відображається**.
   - Якщо значення відсутнє — банер **відображається** повторно.

### 5.3 Дані, що зберігаються

| Ключ              | Значення     | Призначення                       |
|-------------------|-------------|-----------------------------------|
| `cookieConsent`   | `"accepted"` | Стан згоди користувача на Cookie Popup |

### 5.4 Відкликання згоди

Користувач може в будь-який момент відкликати свою згоду, виконавши:

```javascript
localStorage.removeItem('cookieConsent');
```

Після цього банер Cookie Popup буде відображено при наступному відвідуванні.
